Graphical representation

Introduction

Graphical representations of data are fundamental for the understanding of scientific knowledge, as readers often rely on what the experts visually represent in their publications to understand the underlying data-set and interpret their potential scientific meaning. Figures and diagrams not only show the relevant data that support key research findings, but also provide visual information on the interactions among different operations required in scientific reasoning. Being able to adequately and precisely visualize data is also a pillar on which decisions can be made as proposed by different dashboards in the market.

Data visualization has various purposes, such as to make abstract thinking on data series or sets more concrete and (mentally) manipulable, to help readers identify and evaluate some fea- tures of the data, to let users see the possible underlying trends, patterns, processes, mechanisms, etc. of the phenomena considered and studied. The way data are visualized can therefore have important epistemic implications for scientific knowledge, as data visualization is not an "interpretation-free" practice, i.e. a neutral process of data presentation in terms of scientific un- derstanding. There are indeed several ways to transform data into a visual format, each of them entailing different possibilities for data interpretation.

Matplotlib

Introduction

Matplotlib is a library for making two-dimensional plots in Python programming language.

Matplotlib is designed with the philosophy that you should be able to create simple plots with just a few commands, or just one.

NumPy

Introduction

NumPy is a Python library used for scientific computing in Python programming language.

It provides various routines for operations on arrays, including sorting, selecting, I/O, discrete Fourier transforms, basic linear algebra, basic statistical operations, random simulation.

linspace

The linspace command is an in-built function in Python's NumPy library. It is used to create an evenly spaced sequence in a specified interval. 


# Example 1
import numpy as np
x = np.linspace(0, 4 * np.pi, num = 100)
y = np.sin(x)
plt.plot(x, y)
plt.xlabel('x label')
plt.ylabel('sin(x)')
plt.show()

# Example 2 (Drawing two lines simultaneously on a canvas)
import numpy as np
x = np.linspace(0, 4 * np.pi, num = 100)
plt.plot(x, np.sin(x))
plt.plot(x, np.cos(x))
plt.xlabel('x label')
plt.legend() # Place a legend on the Axes
plt.xlim(x[0], x[-1]) # set the x limits of the current axes
plt.show()
Bar Plot

pyplot.bar

A barplot shows the relationship between a numeric and a categoric variable.


import numpy as np
import matplotlib.pyplot as plt

categories = ('Undergraduate Degree', 'Graduate Degree', 'PhD', 'Post-Doc')
students_num = (432, 300, 100, 50)

# np.arange(...) Returns equidistant values within a given range.
# it is equivalent to [x for x in range(4)]
x_pos = np.arange(len(students_num))
plt.bar(x_pos, students_num, align='center')

# set the current tick locations and labels of the x-axis.
plt.xticks(x_pos, categories, rotation=20)
plt.xlabel('Categories')
plt.ylabel("Students number")

plt.show()

Histogram

pyplot.hist


import numpy as np
import matplotlib.pyplot as plt   

# Generate a random samples from a normal (Gaussian) distribution.
x = np.random.normal(loc=0.0, scale=1.0, size=5000)

# n: is the number of counts in each bin of the histogram
# bins: is the left hand edge of each bin
# patchesis the individual patches used to create the histogram, e.g a collection of rectangles
n, bins, patches = plt.hist(x, 20, facecolor='green')

# The patches can be used to change the properties of individual bars as in these examples.
plt.setp(patches[10], 'facecolor', 'y')

plt.xlabel('values')
plt.ylabel('frequency')

plt.show()

Pie Chart 


import matplotlib.pyplot as plt

# Pie chart, where the slices will be ordered and plotted counter-clockwise:
labels = 'Frogs', 'Hogs', 'Dogs', 'Logs'
sizes = [15, 30, 45, 10]
explode = (0, 0.1, 0, 0)  # only "explode" the 2nd slice (i.e. 'Hogs')

# The default startangle is 0, which would start the "Frogs" slice on the positive x-axis.
# This example sets startangle = 90
# such that everything is rotated counter-clockwise by 90 degrees, and the frog slice starts on the positive y-axis.
plt.pie(sizes, explode=explode, labels=labels,
        autopct='%1.1f%%', shadow=True, startangle=90)

plt.show()
